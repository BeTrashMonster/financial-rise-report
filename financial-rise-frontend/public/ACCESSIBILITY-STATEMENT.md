# Accessibility Statement for Financial RISE Report

**Last Updated:** December 22, 2025

## Our Commitment

Financial RISE Report is committed to ensuring digital accessibility for all users, including individuals with disabilities. We strive to continually improve the user experience for everyone and apply the relevant accessibility standards.

## Standards

This application conforms to the **Web Content Accessibility Guidelines (WCAG) 2.1 Level AA**. These guidelines help make web content more accessible to people with disabilities, including:

- Visual impairments (blindness, low vision, color blindness)
- Hearing impairments (deafness, hard of hearing)
- Motor impairments (limited hand/arm function)
- Cognitive and learning disabilities
- Speech disabilities

## Accessibility Features

Our application includes the following accessibility features:

### Keyboard Navigation
- All functionality is accessible using a keyboard alone
- Logical tab order through all interactive elements
- Visible focus indicators on all focusable elements
- No keyboard traps that prevent navigation
- Support for standard keyboard shortcuts (Tab, Enter, Space, Arrow keys)

### Screen Reader Support
- Compatible with NVDA and JAWS screen readers
- Proper ARIA labels on all interactive elements
- Semantic HTML structure with landmarks (header, main, navigation, footer)
- Descriptive headings and labels
- Form validation errors announced to screen readers
- Dynamic content changes communicated via live regions

### Visual Design
- High color contrast ratios exceeding WCAG AA standards (4.5:1 for text, 3:1 for UI components)
- Text resizable up to 200% without loss of functionality
- No information conveyed by color alone
- Clear and consistent visual design
- Minimum font size of 14px (per brand requirements)

### Forms and Error Handling
- All form fields clearly labeled
- Required fields indicated with text and visual markers
- Error messages clearly associated with form fields
- Helpful validation messages
- Confirmation dialogs for destructive actions

### Responsive Design
- Fully responsive across desktop, tablet, and mobile devices
- Touch targets meet minimum size requirements (44x44px)
- Content reflows properly on different screen sizes
- No horizontal scrolling at 320px width

## Testing and Compliance

Our application has been tested using:

- **Automated Tools:** axe-core accessibility scanner
- **Manual Testing:** Keyboard navigation, screen reader testing (NVDA, JAWS)
- **Browser Testing:** Chrome, Firefox, Safari, Edge
- **Device Testing:** Desktop, laptop, tablet, mobile
- **Static Analysis:** ESLint with jsx-a11y plugin
